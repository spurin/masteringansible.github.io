DNS Issues with Lab Virtual Machines

2019-03-13
DNS, Issues, Virtual Machine

An issue that I’ve been contacted about on the course, is a lack of internet connectivity within the guests of the VM lab. Naturally, when a system is started it’s almost like a motor reflex action, to ping google.com :)

Whilst this would normally work, in the lab setup, the dedicated dnsmasq virtual machine acts as a DNS server for the rest of the lab environment. It provides internal DNS resolution for each of the nodes and DNS forwarding, for all other nodes in the lab.

If you’re experiencing issues with DNS resolution, check that the dnsmasq virtual machine is running, that it’s IP address 192.168.0.40 is contactable, and from the dnsmasq guest, that internet connectivity is available.

Should you have DNS issues outside of this, please leave a comment below or contact me via the social media links provided.
